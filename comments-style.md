# Стиль комментариев в коде

Пиши комментарии так, как их писал бы живой разработчик вручную.

Запрещено:
- декоративные разделители: `========`, `--------`, `********` и т.п.
- «плакатные» заголовки в комментариях
- шаблонные AI-комментарии

Комментарии должны быть короткими, естественными, по делу, простым человеческим языком.

Стиль:
- обычные `#` или docstring там, где реально уместно
- без пафоса и канцелярита
- без повторения очевидного
- без фраз «данная функция осуществляет», «ниже реализована логика»

**Хорошо:**
```python
# Загружаем шаблон протокола
# Если базы знаний нет, просто работаем без RAG
# M4A сначала конвертируем в WAV, потом отдаем в Whisper
```

**Плохо:**
```python
# ===================== ЗАГРУЗКА ШАБЛОНА =====================
# ДАННАЯ ФУНКЦИЯ ОСУЩЕСТВЛЯЕТ ЗАГРУЗКУ ШАБЛОНА
# --------- БЛОК РАБОТЫ С RAG ---------
```

Цель: ощущение, что код писал человек для своей рабочей тетради, а не нейросеть.
